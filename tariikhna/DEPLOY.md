# Deploying Tariikhna for free

The app runs as a **single Streamlit app** that reads the bundled story database
(`backend/tariikhna.db`) and illustrations (`backend/media/`) **directly** — both
are committed to the repo. So deploying is **one** free service, **no backend
host, no credit card, no secrets**.

| Part | Host |
|------|------|
| The whole app | **Streamlit Community Cloud** (free) |

> The FastAPI backend is still in the repo (`backend/`) and works locally if you
> want it, but it is **not** needed for deployment — the Streamlit app reads the
> data itself. See "Optional: run with the API backend" at the bottom.

---

## Step 1 — Push the code to GitHub

The app lives in its own repo, **`Mohamad-Shata/Tariikhna_app`**, flat at the root
(`backend/` and `frontend/` are top-level). Commit and push:

```bash
git add -A
git commit -m "Deploy"
git push
```

Make sure `backend/tariikhna.db` and `backend/media/` are committed (they are the
content the app serves — not gitignored). `backend/.env` and the venv stay ignored.

---

## Step 2 — Deploy on Streamlit Community Cloud

1. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
2. **New app** → **Deploy a public app from GitHub**.
3. Settings:
   - **Repository:** `Mohamad-Shata/Tariikhna_app`
   - **Branch:** `main`
   - **Main file path:** `frontend/streamlit_app.py`
4. Click **Deploy**. It installs `frontend/requirements.txt` automatically.

That's it. **No secrets to set** — with no `TARIIKHNA_API_URL` configured, the app
auto-selects local data mode and reads the bundled DB + images. Your app gets a
public URL like `https://tariikhna.streamlit.app`.

---

## Updating the stories later

The data is generated by the importer and committed to the repo, so updating is
"re-import → commit → push", and the app redeploys itself:

```bash
# 1. regenerate the DB + images + audio from the source
cd backend
../.venv/Scripts/python import_storybook.py    # uses the Graduation-Project repo next door
#   (or pass --images / --stories / --audio explicitly — see README)

# 2. ship it
cd ..
git add -A
git commit -m "Update stories"
git push          # → Streamlit Cloud auto-redeploys with the new data
```

(The source files live in the separate `Graduation-Project` repo.)

## Optional: run with the separate API backend

If you specifically want the two-service setup (Streamlit frontend → FastAPI
backend), set `TARIIKHNA_API_URL` and the app switches to API mode automatically.

- **Locally:** start the backend (`cd backend && uvicorn app.main:app --reload`),
  then run the frontend with `TARIIKHNA_API_URL=http://127.0.0.1:8000`.
- **Hosted, card-free:** deploy the backend to a **Hugging Face Docker Space**
  (free, no card), then put `TARIIKHNA_API_URL = "https://<you>-<space>.hf.space"`
  in the Streamlit app's **Settings → Secrets**. (Render also works but now
  requires a card on the free tier.)
